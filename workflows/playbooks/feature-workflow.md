# workflows/playbooks/feature-workflow.md

## 目标
定义标准 feature 任务从提出到交付的完整 SOP。

## 适用范围
适用于：
- 新增功能
- 明确需求的小型迭代
- 可拆解为标准研发流程的 feature 任务

不直接覆盖：
- 严重线上事故
- 紧急 hotfix
- 大型架构重构

## 参与角色
- CEO / 用户
- PM Agent
- Tech Lead Agent
- Dev Agent
- QA / Review Agent
- Ops / Release Agent

## 主流程
1. CEO / 用户创建任务草稿
2. PM 澄清需求并定义验收标准
3. Tech Lead 输出技术方案
4. Dev 实现并自测
5. QA / Review 审核
6. CEO / 用户确认交付范围
7. Ops / Release 收口归档

## 关键停机点
- 需求边界不清
- 多方案待拍板
- 审核未通过但想强推
- 涉及外部影响或高风险动作
