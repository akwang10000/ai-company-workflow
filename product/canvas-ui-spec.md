# product/canvas-ui-spec.md

## 目标

把 workflow 画布从“原则描述”收成“可直接指导前端实现”的界面规格，确保 Codex 能按文档实现第一阶段画布主界面。

---

## 当前文档定位

这份文档负责：

- 画布页内部布局
- 主画布 / 侧栏的信息层级
- 节点 / 连线 / 状态的可视规则
- 画布交互细节

这份文档不负责：

- 重新定义状态机
- 重新定义动作名
- 重新定义 workflow 主链路
- 替代 `product/task-transition-api-and-actions.md`

若与 product 主真相文档冲突，以 product 主真相文档为准。

---

## 设计定位

第一阶段画布的目标不是做复杂 BPMN 工具，而是做：

- 一眼看懂当前流程
- 一键打开节点详情
- 一眼知道卡在哪里
- 一眼看到当前负责人是谁

默认优先：

- 易懂
- 可追踪
- 可演示
- 可轻量操作

不优先：

- 高复杂拖拽编排
- 大量专家参数暴露
- 满屏技能节点

---

## PC 端布局规格

建议采用三栏式布局：

### 1. 左侧栏：导航与视图切换

#### 内容

- 当前 workspace 名称
- 当前 workflow 名称
- 视图切换：
  - 全流程视图
  - 当前任务视图
  - 当前角色视图
- 快捷过滤：
  - 只看等待拍板
  - 只看返工节点
  - 只看当前激活节点

### 2. 中间主区域：Workflow 画布

#### 默认内容

- 节点
- 连线
- 当前节点高亮
- 当前 owner 标识
- 状态标签

#### 画布交互

- 单击节点：选中并在右侧打开详情
- 双击节点：若第一阶段支持编辑，则进入轻量编辑；否则与单击等价
- 拖动画布空白处：平移
- 鼠标滚轮 / 触控板：缩放
- 点击“重置视图”：回到适配屏幕的默认缩放

### 3. 右侧栏：节点详情面板

#### 默认展示

- 节点标题
- 节点类型
- 当前状态
- 当前负责人
- 输入
- 输出
- 风险点
- 当前启用技能
- 最近执行记录摘要
- 可执行动作（若允许）

---

## 手机端布局规格

手机端不做完整三栏，而采用：

- 顶部标题栏
- 中间缩略画布 / 当前流程卡片
- 底部抽屉详情

### 手机端优先能力

- 查看当前流程走到哪
- 查看当前负责人
- 查看 `Waiting Decision`
- 打开节点详情
- 审批拍板

### 手机端不优先

- 复杂模板编辑
- 大范围拖拽布局
- 大量节点同时精细编辑

---

## 节点类型与视觉规则

### 1. 角色节点

表示 `pm_agent / tech_lead_agent / dev_agent / qa_review_agent / ops_release_agent` 等执行角色。

### 2. 任务节点

表示某个任务或阶段任务承载点。

### 3. 审批节点

表示 `Waiting Decision / Approver` 决策门。

### 4. 状态节点

表示 `Draft / Ready / In Progress / Waiting Handoff / Waiting Decision / In Review / Rework Required / Ready for Delivery / Done / Archived / Cancelled` 等状态位。

---

## 连线类型与样式规则

### 1. 标准流转线

- 颜色：主色 / 中性色
- 含义：正常推进

### 2. 返工线

- 颜色：警示色
- 含义：`Rework Required`

### 3. 决策线

- 颜色：强调色
- 含义：进入 `Waiting Decision` 或从决策节点流出

### 4. 当前激活路径

- 样式：加粗 / 高亮
- 含义：当前任务正在走的主路径

---

## 节点状态可视规则

建议使用“颜色 + 标签 + 轻角标”三层组合，不只靠颜色。

### 示例

- `Draft`：灰
- `Ready`：蓝灰
- `In Progress`：主色
- `Waiting Handoff`：黄 / 琥珀
- `Waiting Decision`：红 / 强调
- `In Review`：紫
- `Rework Required`：橙
- `Ready for Delivery`：青 / 绿色过渡
- `Done`：绿
- `Archived`：灰化
- `Cancelled`：深灰 / 斜纹

---

## 信息层级规则

### 主画布上只放

- 节点名称
- 节点类型
- 当前状态
- 当前负责人（简版）
- 是否激活 / 卡住

### 右侧详情再放

- 输入输出
- 风险
- 技能
- 交接说明
- 执行日志
- 审批建议

### 不要直接堆在主画布上的内容

- 长文本 handoff
- 大段日志
- 技能参数细节
- schema / timeout / retry 等内部参数

---

## 顶部工具栏建议

PC 端画布页顶部建议至少有：

- 返回工作空间
- 当前 workflow 标题
- 视图切换
- 搜索节点
- 缩放控制
- 重置视图
- 过滤开关

第一阶段不一定要有：

- 导出图片
- 自由编排模式
- 批量编辑节点

---

## 第一阶段支持的操作

- 查看 workflow 画布
- 查看当前节点
- 查看当前 owner
- 查看节点详情
- 查看是否卡在 `Waiting Decision / Rework Required`
- 从节点详情进入相关任务或审批页

## 第一阶段暂不强求的操作

- 任意拖拽重排节点
- 复杂多选编辑
- 节点级自由样式配置
