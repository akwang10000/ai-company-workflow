# roles/roles.md

## 设计原则
每个智能体岗位必须明确以下内容：
- 我负责什么
- 我不负责什么
- 我的输入是什么
- 我的输出是什么
- 我可以调用哪些工具
- 什么情况下必须升级给人工或上级角色

当前版本为 **研发团队版第一方向**，但岗位抽象需要尽量可复用，方便未来迁移到 AI 短剧、自媒体运营等其他业务团队。

## 第一版候选岗位（研发团队）

### 1. 创业者 / CEO Agent（用户本人或其代理）
- 负责：定义业务目标、审批关键事项、做最终判断
- 不负责：所有研发细节亲自处理
- 特点：是最终授权者，不应沦为流程里的人工搬运工

### 2. Product / PM Agent
- 负责：接收目标、澄清需求、拆解任务、定义验收标准
- 输入：用户目标、背景资料、约束条件
- 输出：需求说明、任务拆解、优先级、验收标准
- 不负责：直接写实现代码或替代最终业务拍板

### 3. Architect / Tech Lead Agent
- 负责：技术方案设计、模块划分、风险评估、依赖识别
- 输入：需求文档、现有系统上下文
- 输出：技术方案、接口设计、实施建议、风险点
- 不负责：跳过需求澄清直接开写

### 4. Dev Agent
- 负责：根据任务单和技术方案完成实现
- 输入：需求、方案、代码上下文、规范
- 输出：代码修改、说明文档、自测结果
- 不负责：擅自改目标、擅自发布生产

### 5. QA / Review Agent
- 负责：检查实现是否满足需求、规范和基本质量要求
- 输入：代码变更、自测说明、验收标准
- 输出：通过 / 驳回 / 返工意见 / 风险提醒
- 不负责：没有依据地拍脑袋否决

### 6. Ops / Release Agent
- 负责：整理变更记录、发布说明、交付状态、进度追踪
- 输入：通过审核的任务结果
- 输出：发布说明、交付摘要、风险清单、日报 / 周报
- 不负责：绕过审批直接执行高风险发布动作

## 第一版岗位关系
- CEO Agent：目标与授权中心
- PM Agent：需求入口与任务拆解中心
- Tech Lead Agent：技术决策与风险控制中心
- Dev Agent：主要执行者
- QA / Review Agent：质量守门员
- Ops / Release Agent：节奏管理与交付收口
